## Archive Guidelines

**When to Archive:**
- Documents that are no longer actively referenced
- Proposals that have been implemented or superseded
- Historical planning documents after phase completion
- Outdated specifications or design docs

**When NOT to Archive:**
- Active sprint/phase documentation
- Current architecture or design documents
- README files and getting started guides
- Test plans still in use

**Retention Policy:**
- Keep archived docs indefinitely for project history
- Add a brief summary when archiving (like this README)
- Maintain original file structure and naming
